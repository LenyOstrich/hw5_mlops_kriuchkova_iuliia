# Домашняя работа: инструменты MLOps (Marimo, MLflow, DVC, Feast, Diagrams)
В рамках данной работы были изучены и применены современные инструменты MLOps для построения воспроизводимого и наблюдаемого ML-пайплайна. Основной акцент сделан на экспериментальную работу, управление данными, трекинг экспериментов и визуализацию архитектуры системы.

## Использованные инструменты
* Marimo — интерактивная среда для разработки ноутбуков
* Google Colab — облачная среда для выполнения экспериментов
* MLflow — трекинг экспериментов и метрик моделей
* DVC (Data Version Control) — обеспечение воспроизводимости экспериментов
* Feast — feature store для управления признаками
* Diagrams (Python library) — построение архитектурных схем системы

## Выполненные задачи
1. Сравнение Marimo и Google Colab
* Изучены особенности Marimo как реактивной альтернативы Jupyter/Colab
* Проведено сравнение с Google Colab по удобству, интерактивности и воспроизводимости
* Сделаны выводы о преимуществах Marimo в реактивных вычислениях и управлении зависимостями между ячейками
2. MLflow: трекинг эксперимента
* Запущен эксперимент в MLflow
* Зафиксированы параметры модели и метрики качества
* Проверен корректный расчёт метрики (MSE)
* Результаты эксперимента доступны в MLflow UI
3. Воспроизводимость с помощью DVC
* Инициализирован Git- и DVC-репозиторий
* Настроено версионирование данных и пайплайна
* Обеспечена возможность повторного запуска эксперимента с теми же входными данными
* Проверена корректность работы DVC pipeline
4. Feature Store (Feast)
* Создан локальный feature store
* Описаны сущности и фичи в feature_store.yaml
* Настроено хранилище признаков для последующего использования в моделях
* Проверена работа конфигурации и структуры проекта
5. Архитектура системы (Diagrams)
* Построена архитектурная схема ML-системы с использованием библиотеки Diagrams
* Отображены ключевые компоненты
* Диаграмма используется для визуализации полного ML-пайплайна

## Как запустить проект

Для всех ячеек кроме DVC ноутбук HW5_MLOps_Крючкова_Юлия.ipynb необходимо запустить в Colab.
Для DVC пайплайна достаточно запусить ячейку с install dvc и далее скрипт для инициализации DVC и Git, создания коммита и добавления всех необходимых файлов в коммит.
